# Core Loop

## The Three Layers

```
SESSION
  └─ Player acts in the living world
       (explore, survive, build relations, accumulate)
            ↓
DEATH / SUCCESSION
  └─ Choose heir or accept random stranger
       World state persists and continues simulating
            ↓
LEGACY
  └─ Prior run's mark on the world shapes new run
       (reputation, ruins, changed factions, inherited resources)
            ↑
       (loop)
```

---

## Session Layer (within a life)
- Survive in a medieval world with real simulation pressures
- Build wealth, reputation, relationships, knowledge
- Make choices that ripple outward into the living world

## Succession Layer (at death)
- **Option A:** Heir (child or named ally) — inherits assets, reputation modifier, location context
- **Option B:** Fresh stranger — starts cold but world carries your marks
- See [[Legacy System]] for full mechanic

## Legacy Layer (across lives)
- World state is never reset between runs
- Prior character's actions leave physical and social traces
- Accumulating or destroying is permanent at the world level
